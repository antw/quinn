Quinn – a jQuery slider library

Quinn is a jQuery plugin which creates sliders (aka ranges, aka track bars) for HTML-based application. The project is hosted on Github. You can report bugs and discuss feature on the issues page, or direct your tweets at @antw.

Quinn was developed for Quintel Intelligence.

Options

In order to customise the Quinn slider, you may pass extra options when initializing: $(selector).quinn(optionsHash);

Minima and Maxima range: [min, max]

By default, the lowest value of a Quinn slider is 0, and the highest is 100. By supplying a range option, you can change these to any values you wish. range should be a two-element array in the format [minimum, maximum]. 

$('.slider').quinn({ range: [0, 11] });

Initial Values value: number

Since you probably don’t want to initialize the slider at the lowest possible value each time, the value option allows you to supply a value which should be used instead. 

$('.slider').quinn({ value: 25 });

Intervals interval: number

The interval changes the slider so that only values which are divisble by the interval may be chosen. If the slider is initialized with a custom value (see above) which is not evenly divisible by the interval, it will be rounded to the nearest acceptable value. 

$('.slider').quinn({ interval: 10, value: 8 });

Combined with the range option, interval can be powerful: 

$('.slider').quinn({ range: [0, 1.21], interval: 0.01 });

Selectable Ranges selectable: [min, max]

Sometimes it is desirable that only part of the slider may be selected by the user. In this case, the selectable option may be passed when creating the slider instance. 

$('.slider').quinn({ selectable: [35, 80] });

When using selectable, your interval settings will still be respected, and selectable values which are not at one of the interval values will be rounded to the nearest inclusive interval. Note that in the example below the lowest selectable value 35 is not available since interval: 20 is used. The lower value is instead rounded to 40. 

$('.slider').quinn({ selectable: [35, 80], interval: 20 });

Effects effects: bool  and  effectSpeed: number

Some actions make use of jQuery effects. For example, clicking anywhere on the slider bar immediately changes the slider value but, by default, animates movement of the handle. Animations may be disabled entirely through us of the effects: false, option, or the duration of the animation changes with the effectSpeed option.

If the Easing library is available, your default animation easing function will be used. 

$('.slider').quinn({ effects: false });

Callbacks

Callbacks are supplied when calling jQuery.quinn just like any other option.

onSetup onSetup: function (currentValue, quinn)

onSetup is run only once; immediately after the Quinn constructor has completed. Two arguments are supplied: the current value (which may be different to the initial value you supplied, if your value was not value), and the Quinn instance.

onChange onChange: function (newValue, quinn)

The onChange callback is run each time the slider value changes. The function is supplied with two arguments: the new slider value, and the Quinn instance the previous slider value can be retrieved with quinn.value since the value attribute is only update after the callback has completed. 

$('.slider').quinn({
    onChange: function (newValue, slider) {
        var h = (128 - newValue * 1.28).toFixed(),
            l = (35 + newValue * 0.1).toFixed();

        $('.onchange-example .value').
            css('color', 'hsl(' + h + ', 50%, ' + l + '%)');
    },

    onSetup: function (value, slider) {
        // Run the onChange callback immediately.
        slider.options.onChange.call(slider, value);
    }
});

By explicitly returning false in your callback, you can cancel the slider value change. 

$('.slider').quinn({
    onChange: function (newValue, slider) {
        // Prevent selection of 41 to 59.
        if (newValue > 40 && newValue < 60) {
            return false;
        }
    }
});
